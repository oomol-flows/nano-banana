## 工作原理

Nano Banana为您提供了可以无缝协作的构建模块:

### 从文本生成图片

**图像生成器**模块将您的描述转化为真实图片。您可以:
- 用简单的语言描述您想创作的内容
- 选择生成多少个变体(非常适合探索不同选项)
- 选择您偏好的图像格式(JPEG、PNG 或 WebP)
- 为您的需求选择完美的尺寸(方形、宽屏、竖版等等)

系统会处理您的描述,并在图片准备好后自动交付给您。

### 编辑现有图片

**开始编辑:**
**编辑请求**模块是您编辑之旅的起点。您需要提供:
- 想要编辑的图片(使用网络链接)
- 用日常语言描述您想要的修改

AI会处理您的请求并给您一个追踪编号,用于查看结果。

**获取结果:**
**获取结果**模块会在您编辑或生成的图片准备好后取回它们。它会自动检查图片是否完成并交付给您。您可以调整:
- 检查更新的频率(适合大批量处理)
- 超时前等待的时长

### 完整工作流

**图像生成器工作流:**
将图像生成的所有步骤合并成一个流畅的过程。您只需描述想要的内容、选择偏好设置,它就会处理一切 - 创建图像并交付最终结果。

**图像编辑器工作流:**
将两个编辑步骤合并成一个无缝的过程。您只需提供图片和说明,它会处理一切 - 提交请求、等待处理、交付成品图片。

## 模块组合使用

这些构建模块设计上可以和谐配合。以下是一些创意组合方式:

**先生成后编辑:**
从文本创建图片,然后进行微调编辑。例如,生成"一个现代客厅",然后编辑它"在窗边添加植物"。

**批量创作:**
用相同的描述一次生成多个变体图片,然后选择您最喜欢的进一步编辑。

**迭代优化:**
从生成的图片开始,编辑它,然后用该结果作为灵感生成更多变体或进行额外编辑。

**结合云存储:**
生成或编辑图片,然后与云存储模块结合,自动整理和保存您的作品。

**自定义工作流:**
当您需要精确控制时间、想同时处理多个请求或构建更复杂的图像处理流程时,可以分别使用单独的构建模块。

## 获得最佳效果的技巧

**对于生成:**
- **描述要详细:** "带有白色橱柜、大理石台面和自然光的现代厨房"比只说"一个厨房"效果更好
- **包含风格细节:** 提及艺术风格、情绪或氛围:"水彩风格"或"带有戏剧性光照"
- **指定关键元素:** 包括关于颜色、构图或特定物体的重要细节
- **尝试不同选项:** 试试不同的宽高比,生成多个变体以探索可能性

**对于编辑:**
- **具体明确:** "在前景添加一朵红玫瑰"比只说"加一朵花"效果更好
- **一次一个变化:** 对于复杂的转换,可以分步骤进行
- **保持自然:** 要求的修改要与现有图片自然契合
- **批量处理相似编辑:** 将需要类似修改的多张图片一起处理,提高效率
